Example 5: different ways to define colours

This example shows that colours can be defined in multiple ways, as in the CSS the names for each will be different colours and use the respective methods. The methods are Red/Green/Blue, hex code, Hue/Saturation/Lightness, Red/Green/Blue/Alpha, Hue/Saturation/Lightness/Alpha, and colour name. Colour can be applied to text, the background, to a border, or to many other things for different effects.

  • Red/Green/Blue (or RGB) works by providing a value from 0 to 255 for the channels of red green and blue, which is how light combines into various colours as those are the primary colours of light (or "additive colour mixing"). This is also used by Red/Green/Blue/Alpha (or RGBA) but the added channel "Alpha" is used for transparency, with a value from 0.0 (completely transparent) to 1.0 (fully opaque). The alpha channel can be seen in action when Example 16.1 overlaps with another example, as you will be able to see through it.
  • Hex code is one of the most used ways to describe colour for computers. It is seen as a # symbol followed by 6 "digits" of haxadecimal (numbers form 0-9 and letters from a-f to represent numbers from 10-15, with 10 in hexadecimal being the same as 16 in decimal). Each group of 2 digits is used for the same channels as RGB in order. So since the colour being used to represent hex codes in this example is #3aefeb it is equivalent to an RGB value of [58,239,235] which makes it the light blue/green colour that you see. Hex codes are also able to use the alpha channel by adding another 2 digits to the code, with higher values being more opaque and lower values being more transparent.
  • Hue/Saturation/Lightness (HSL) and Hue/Saturation/Lightness/Alpha (HSLA) are in contrast less widely used. The value for Hue goes through each colour ranging from 0 to 359 (0 is the same as 360, being red, and all other values are a different part of the light spectrum), and the Saturation and Lightness values range from 0% to 100%. HSLA is like RGBA in which it also adds in the Alpha channel, with the same value range.
  • The web also has 140 colour names that are supported by all browsers, which are symbols pointing to predefined hex code values. The colour used to represent colour names for this example is magenta, which means it is the equivalent of #ff00ff.

Example 6: background images

This example shows that it is also possible to use an image for a background in a container instead of just a colour.

The background image is also capable of being repeated (such as for a tileable pattern you want to use for a given box or container) or it can be set to not do so. It is also possible to only set it to repeat either horizontally or vertically.

If the background is set to not be repeated, then the rest of the backgorund will be set to look the same as the background behind the element being assigned an image, unless it is also given a background colour, which will show behind the image so a background colour would only show if the image is either transparent or doesn't fill the whole box similar to how this one has a sunset that doesn't fill the box.

There is also a property for changing the size of the background, so it could make a background image match the size of the element it is assigned to. This is set to "contain" in this example so that the image is made larger so that its height matches the container while keeping the same proportions so the width doesn't make it look squashed or stretched.

This example also gives the text a dark shadow so that it is somewhat easier to read with the image behind it.

There are various types of lists available for use in a web page, and CSS can style them in various ways. Here we have an "Ordered List", marked to show an order:

  1. This list has been styled so that it is marked with roman numerals, as opposed to the default numbers.

  5. "lower-alpha", lower case use of english letters.
  6. "lower-greek", lower case use of greek letters, such as α, β, γ, δ, etc.
  7. "lower-latin", which displays the same as lower-alpha.
  8. "lower-roman", which uses roman numerals with the letters used in lower case (such as i, ii, iii, iv).
  9. "upper-alpha", upper case use of english letters.
  10. "upper-greek", upper case use of greek letters, such as Α, Β, Γ, Δ, etc.
  11. "upper-latin", which displays the same as upper-alpha.
  12. "upper-roman", which uses roman numerals with the letters used in upper case (such as I, II, III, IV).

Example 16: Position

The position property can be used to change where different elements appear within their container. For most elements that this would be used for, it would define their position within the window itself, but for this example there is a box within this example's container which will contain a different element for each position.
Each box contains text stating what the position value is for that box.

  • The static value is the default position value, and causes elements to follow the natural flow of the page so each one is shown on top of each other in the order they are in the HTML, and causes elements to not be affected by the other "positioning properties" (top, bottom, left, or right). It also always shows "behind" overlapped elements, even if it is placed after them in the HTML, this being because it doesn't count as "positioned".
  • The relative value is the same as static, except that the element can be affected by positioning properties, as can be seen with the blue box in the example area.
  • The fixed position, instead of causing the element to follow the page, uses the positioning properties to show the element as fixed to the viewport, not moving when you scroll up or down. This can be seen with Example 16.1, which has the transparent green background (so that you can still see what is behind it) and has been in the top right of the screen instead of inside the example area.
  • The absolute position is much like Fixed, except that instead of being relative to the viewport, it is positioned relative to the nearest positioned ancestor, meaning that if it is within an element which has a position property other than static (which the example area has a position property of relative, with no positioning properties used) it uses that element for the positioning properties. You will see the yellow box is in the bottom right of the example area.
    If there was no positioned ancestor, it would instead use the document body, and move with page scrolling, so if the example area was static then the yellow box would be in the bottom right if the page was scrolled all the way to the top.
  • The sticky position acts like the relative position, but if it is given a top positioning property then when you scroll past it then it sticks to the viewport using that positioning property for the distance from the top while staying within the parent container (and not moving past the bottom of it).

Example 18: Display Property

The display property is the first piece of CSS that won't be given visual examples due to being mostly for full pages, but this example will still give details on each form of display.

  • The deafault display of an element is either block, or inline.
  • Block elements (by default) take up the full available width and any elements after them are placed immediately below them, with some examples of elements that are by default "block-level" are the div, the header elements (h1 to h6), p, and so on.
  • Inline, in contrast, doesn't start a new line and only takes up as much space as is required for the content within it. some examples are the span and a elements, which can be used within paragraphs of text to either change the CSS styling of individual words or phrases with span, or turn specific words (like a company name) into a link to another page.
  • Using CSS, it is possible to change the display property, with a number of options. One option is "none", which hides the element and can be used with a button and javascript to hide and show the element by changing it between "display: none;" and some other display value. This can be done similarly using "visibility: hidden;", however instead of it being as if the element was never there, it instead just makes it invisible which means the space it would have taken up shows as blank.
  • There are also display values for various layouts such as grid, flex, and similar which can be both block-level and inline-level and confer different ways that it can be displayed such as allowing or disallowing different CSS properties from altering the element.
